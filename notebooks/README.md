# Notebooks

The notebook set is intentionally split by audience.

## Start Here

- `notebooks/01_reference_api_happy_path.ipynb`: beginner-oriented walkthrough of the OSS reference API flow

## Advanced / Validation Grade

- `notebooks/02_tmp_data_full_api_tutorial.ipynb`: heavier end-to-end validation notebook for real `tmp_data`, optional native packages, and broader feature exploration

The `02_...` notebook is kept for advanced validation because it assumes more
context, more data, and a more evaluator-like setup than a first-run OSS user
should need.
